Class Overview

The BaseMeasurement is the base implementation of the Measurement, and wraps wraps an instance of a Unit, and the value it returns is always in terms of this Unit. The Unit wrapper might seem annoying at first, but it is critical to avoid misinterpreting the unit and crashing your lander into Mars (http://en.wikipedia.org/wiki/Mars_Climate_Orbiter). Most applications will not use this class directly, but will import specific child classes that correspond to specific types of measurements - i.e. the parameterized instances of this class with a Unit. That may seem like a "psuedo-typedef" but we're using it it to enforce the binding between the measurement and its unit type. This unfortunately means we have to add constructors to every child class because they aren't inherited from Measurement. If you know of a better way, please say so.
